Welcome!

This desktop application runs at home while you are out on a hike. It starts sending alert e-mails if you are not back within the scheduled time. In case you have an accident in an area with no phone connection, this might save your life!

You need to have a mail-provider and -account, and a computer with Internet connection. Further you need mail contacts, at least one, that could organize help for you in case of an emergency.

  • None of your contacts will receive an e-mail when you are back in time and press the "Home Again" button.

  • But when you are overdue, the first contact will get an e-mail (with a text you are going to work out now).

  • If a contact does not respond within an hour, the next contact will get the same e-mail, and the previous will be notified about that shift. Of course all of this is configurable.

  • When one of the contacts responds to the alert e-mail, no more contacts will be notified.

  • The application uses a unique MAIL-ID to distinguish alert responses from your private e-mails.

On this first page you can choose your language, or load a hike-file in case you already created hikes before.

Next time you start this application, you most likely will not see this page any more, but you can always navigate "Back" to it and load old hikes. They get stored in JSON format by default in

$HOME/hiking-emergency-alert/hike.json

You can change that default by running the application with -Dhike.home=MyDirectory and -Dhike.file=MyFile.json.

You can also change the font-size between 80 and 140 percent, using -Dhike.fontPercent=120.

This is an open-source application that does not leak or steal your data, although it needs Internet connection to send alert e-mails and receive response e-mails. Your e-mail provider password will not be stored anywhere, you have to input it manually on every hike start. You can check the integrity of the source-code on github.com / fritzthecap / hiking-emergency-alert.
